# 身份、事件与授权

## 分析主键

| 层级 | 含义 | 用途 |
|---|---|---|
| Account | 单平台/店铺账号 | 平台内行为 |
| Person | 经授权且可合理归并的个人 | 生命周期与触达 |
| Household | 共享购买决策的家庭/组织 | 避免重复获客/权益 |

确定性规则优先，概率匹配仅形成带置信度的候选边。低于业务审定阈值不合并。保留原账号与合并记录，使错误合并可逆。

Household 关系用于实验防污染、重复获客评估和家庭购买分析，不自动合并为 Person，不继承成员授权、敏感事件、触达资格或权益资格。地址、设备和组织域只能在当前用途获准时参与候选匹配。

## 标准事件

```text
event_id / event_time / ingest_time / customer_key / anonymous_key
market / channel / store / session / event_type / object_type / object_id
quantity / amount / currency / consent_state / source / data_quality_flag
```

事件域包括认知、考虑、交易、履约、使用、服务、反馈和触达。状态快照、训练和实验必须按 `event_time` 重建；迟到数据追加修订版本，不静默改写历史。

## 授权门槛

对每个用途记录授权范围、市场、渠道、取得/撤回时间和保留期。分析可用不等于可触达；撤回、退订、删除和敏感事件优先于模型分数。

## 模块执行协议

### 对象与问题

- 固定客户/账户/家庭粒度、市场、渠道、商品、生命周期、业务截点和用途授权。
- 明确本模块要改变的决策，以及“不分析”会造成的损失或风险。
- 分开观察窗、预测窗、实验窗和结果成熟窗；不可用同一日期代替。

### 机制与计算

- 从事件生成机制、选择机制、处理机制和结果机制逐层解释，不从相关性跳到动作。
- 比例同时输出分子、分母、窗口、缺失、右删失和区间。
- 金额使用贡献利润，扣退款、权益、服务、渠道和风险成本；销售额不等于价值。
- 预测需给时间外校准；因果需有真实分配、曝光、对照和干扰判断。
- 结构化计算保存脚本版本、输入哈希、输出哈希和不可计算原因。

### 约束与证据

- 区分 observed、authorized、derived、inferred、causal；只有合格设计才能升级 causal。
- 授权、退订、删除、敏感事件、频控、公平性、库存和预算为不可补偿约束。
- 缺失不是零值；未成熟不是失败；未曝光不能进入已处理样本。
- 保留反对证据、负向样本、选择偏差和至少一个替代解释。

### 反事实与失败

- 至少比较不触达、当前策略、推荐策略和一个低成本替代。
- 检查自然恢复、回归均值、季节、渠道混杂、跨设备身份错配和长期伤害。
- 身份、收入、授权或曝光不可对账时降级分析层，不让模型吸收异常。
- 工具部分失败时保留已验证事实，相关结论标记 inconclusive，不补行业平均。

### 动作闭环

- 动作写明对象、资格、成本上限、渠道、频次、责任人和审批。
- 同时给成功指标、机制指标、护栏、观察窗、停止条件和回滚路径。
- “不触达”是正式候选；未入选动作必须说明净增量价值或约束原因。
- 回填真实曝光、成本、短期/长期结果、投诉和撤回，区分模型与策略失效。
- 结果只用于声明的允许用途；禁止外推到未覆盖市场、客户或生命周期。

## identity-event-consent-model 专属决策内核

| 维度 | 本模块要求 |
|---|---|
| 核心机制 | 身份图谱与授权图谱分离，边可逆且按用途生效 |
| 计算或判定 | 匹配置信度、事件时间/入库时间、授权取得/撤回时间分别保存 |
| 主要失效 | 共享设备/地址误合并、删除传播缺失、分析授权冒充触达授权 |
| 决策动作 | 低置信不合并；撤回后冻结触达并重算频控、实验和价值 |

本节是该模块独有验收，不得由其他reference的通用协议替代。输出必须逐项引用本节机制，并给出至少一个与本模块失效模式对应的反证或失败测试。
