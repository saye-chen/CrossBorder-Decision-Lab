# 数据契约与自动化执行

## 标准对象

分开存储身份节点/边、授权用途、客户事件、订单账本、VOC 证据/编码、状态快照、模型评分、实验资格/分配/曝光/结果、动作候选/决定/执行和长期结果。所有对象包含稳定 ID、业务时间、入库时间、来源、市场、模型/规则版本和修订记录。

## 执行链路

`授权读取/导入 → 模式与最小字段校验 → 身份/授权用途检查 → 事件和收入守恒 → 分析截点快照 → 描述/预测/因果计算 → 资格与策略约束 → 人工审批 → 外部执行 → 真实曝光/成本/长期结果回收`。

本 Skill 默认只读并输出建议。未经用户明确授权，不写 CRM/广告/客服系统、不发送消息、不发券、不删除客户数据、不创建周期任务。外部执行必须回传动作版本、资格、分配、真实曝光、成本、失败状态、主要/护栏/长期结果；缺少真实曝光时不声称增量。

## 失败与版本

字段缺失、币种不明、身份冲突、授权不明、收入不守恒、实验 SRM、曝光缺失或模型版本不兼容都形成显式失败状态，不用默认值静默补齐。数据修订追加版本，历史报告和决定不静默覆盖。

自动化质量至少监控读取成功率、字段缺失率、身份误合并/拆分、授权拒绝、收入差异、实验曝光率、动作执行失败、模型漂移、策略护栏和删除传播完成率。

## 统一字段补充

每条记录必须含 object_id 与 as_of_time；缺失和零值分开；任务按幂等键重跑；外部写入需明确授权。

## data-contract-and-automation 专属决策内核

| 维度 | 本模块要求 |
|---|---|
| 核心机制 | 对象、事件、授权、策略和结果使用稳定键与双时间 |
| 计算或判定 | 幂等写入按object_id+event_id+version，迟到数据追加修订 |
| 主要失效 | 缺失填零、重复事件、外部写入无审批、删除未传播 |
| 决策动作 | 字段级失败返回partial并阻断受影响Claim |

本节是该模块独有验收，不得由其他reference的通用协议替代。输出必须逐项引用本节机制，并给出至少一个与本模块失效模式对应的反证或失败测试。
