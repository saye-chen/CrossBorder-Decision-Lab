# Skill 联动协议

本 Skill 先完成事实确认、异动归因和情报事件卡，再根据专业问题建议下游路由。下游 Skill 不可用于反向编造事件证据。联动是可选的，每个 Skill 必须能独立运行。

| 情报信号 | 路由 | 输入/回收 |
|---|---|---|
| 新竞品、HHI/竞争环境变化、持续降价 | `category-investment-decision` | 输入事件卡；回收门槛/评分建议调整 |
| 用户要求最终进入、投资或 Go/No-Go | `category-investment-decision` | 输入商品识别卡、竞品池、竞争姿态、证据/假设账本；回收完整投决结论 |
| 稳定差评主题或产品缺口 | `category-investment-decision` VOC | 输入样本范围和链接；回收产品机会与验证 |
| 视频/广告素材换版并伴随效果信号 | `video-link-breakdown` | 输入素材、前后快照、产品一致性；回收内容机制、迁移性与测试假设 |
| Listing 承诺、图片或结构改变 | Listing 专业能力 | 输入变化与证据；回收我方表达调整方案 |
| 利润、广告、库存或供应链冲击 | 经营诊断/对应专业能力 | 输入影响范围和窗口；回收动作、风险与停止条件 |

## 回写保护

只在用户明确提供上游报告或要求回写时联动。回写必须保留原 Skill/模型版本、原结论或分数、新证据/假设 ID、建议调整、置信度和是否重判门槛。不静默覆盖历史报告，未经用户确认不修改旧结论。

## 统一交换补充

CIM拥有竞争事实主权。结果卡显式列允许用途、禁止用途；部分失败保留事实，冲突按证据和主权裁决。

## skill-integration-protocol 专属决策内核

| 维度 | 本模块要求 |
|---|---|
| 核心机制 | CIM只拥有外部对象、变化和竞争归因，输入输出按Claim交换 |
| 计算或判定 | 卡片键=object+snapshot/event+runtime+as_of+fingerprint |
| 主要失效 | 公开代理越权改预算/投资/页面，部分失败被静默补齐 |
| 决策动作 | 按主权与证据裁决；失败Claim降级并保留可验证事实 |

本节必须由该模块自己的证据和失败测试验证，不得用通用附录或其他reference代替。
