# 竞品画像与快照模式

## 核心对象

| 对象 | 必要字段 |
|---|---|
| 自有商品 | `own_product_id`、输入类型/来源、平台、市场、URL/商品 ID、品类、核心任务、客群、场景、价格带、关键属性、差异点、识别置信度、假设 |
| 竞品主体 | `competitor_id`、品牌/卖家、国家、渠道、经营背景 |
| 竞品商品 | `product_id`、`competitor_id`、平台、市场、URL、SKU/ASIN、品类、价格带、上架时间、发现查询词/渠道、纳入理由 |
| 监控快照 | `snapshot_at`、`product_id`、币种、价格、促销、排名、评分/评论、变体、库存信号、Listing/素材版本、来源 |
| 异动事件 | `event_id`、类型、前值/后值、幅度、持续期、证据、归因、置信度、影响 |
| 经营机会 | 对象、窗口类型/时长、建议动作、优先级、验证和停止条件 |

每个竞品商品只归属一个主体；每个快照只关联一个商品。新快照追加而不覆盖旧值。自有商品识别发生修订时保留原值、修订值、证据和时间，不静默覆盖。

## 画像维度

- 经营背景：主体、市场、渠道、经营时长和规模等级。
- 商品矩阵：SKU 宽度、价格带、主力/引流/利润款、上新节奏。
- 内容策略：主图、视频、A+/详情页、核心卖点与承诺。
- 评价结构：数量、增量、星级、差评主题和 QA。
- 运营节奏：促销、广告位、库存、社媒和上下架。

T1 画像完整度目标 ≥80%，T2 ≥60%。缺失字段必须保留为 `null` 并记录原因，不用推测值填补。
