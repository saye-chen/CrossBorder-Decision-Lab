# 扩展竞争信号域协议

## 信号版图与主权

CIM 对外部竞争事实、变化确认、竞争归因和机会/威胁窗口拥有主权；不拥有最终投资、调价、投放、达人合作、补货或品牌动作主权。所有信号必须固定到对象、国家、平台、渠道、观察窗和基线。

| 信号域 | 必监对象 | 允许判断 | 路由 |
|---|---|---|---|
| 广告竞争 | 广告位、品牌/品类词、素材量、节奏代理 | 压力、素材换代、获客成本上行风险 | D09、CIDM |
| 达人竞争 | 合作密度、层级、重复内容、授权投流 | 资源拥挤、合作模式、扩散速度 | D10、D11 |
| 价格促销 | 标价、券、折扣、套装、会员价、活动节奏 | 价格架构变化、价格战、利润挤压 | D06、D12 |
| 渠道竞争 | 平台、DTC、零售、分销、区域进入 | 渠道迁移、冲突与空白 | D08、CIDM |
| 供应物流 | 缺货、交期承诺、仓配速度、包装、变体 | 供应/履约优势与缺货窗口 | D04、D07 |
| 品牌口碑 | 搜索可见性、评论主题、叙事、服务承诺 | 心智变化与承诺—体验落差 | D12、D13 |

## 证据与变化确认

区分 `直接可见事实 / 用户授权后台事实 / 外部来源陈述 / 估算 / 推断`。单次快照只能证明当前可见状态，不能证明趋势。变化事件至少包含前后快照、采集时间、采集方法、对象匹配置信度、基线、变化幅度、替代解释和证据指纹。

广告量、搜索可见性、达人内容量、库存提示和交期承诺均是代理信号。除非有平台后台或可信独立证据，不得写成广告花费、品牌份额、真实销量、达人合同或实际库存事实。

高影响归因至少需要时间顺序、机制一致性和一个独立佐证；否则标为“竞争假设”。涉及欺诈、侵权、操纵评价或违法等高风险指控时，缺少直接且独立证据必须阻断指控。

## 统一事件卡

每个确认事件输出：

1. 对象、市场、平台/渠道、生命周期和基线窗口；
2. 变化事实、证据 ID、截止时间和数据质量；
3. 影响范围、持续性、置信度和替代解释；
4. 对价格、获客、供应、履约、品牌或客户体验的可能机制；
5. `proposed / validated / blocked / inconclusive`；
6. 允许用途、禁止用途、下游主权域和复核时点；
7. 验证动作、成功条件、失效/停止条件。

## 监控节奏与异常

频率由信号半衰期和决策损失决定：活动/广告/缺货可高频，渠道扩张/品牌叙事可周月级。对页面改版、地区差异、个性化展示、登录状态、抓取失败和平台实验分别标记；不得把采集异常当竞争变化。

冲突时保留多源证据：价格下降但套装变化、广告增多但自然可见性下降、评论改善但投诉上升均不得压缩成单一利好/利空。达到持续性门槛后再路由；紧急合规、安全或供应中断信号可先发阻断卡，随后补证。

## 下游边界

- 向 CIDM：竞争姿态、窗口、威胁和资本重评触发，不给最终 Go/No-Go。
- 向广告域：竞争压力与素材/词变化，不给预算或出价。
- 向达人/内容域：合作拥挤与内容扩散，不推断合同或内容效果。
- 向定价/营销域：价格架构与活动节奏，不直接建议调价。
- 向供应链/物流域：可见交期、缺货和包装信号，不声称真实产能库存。
- 向品牌/CIG：外部口碑与承诺落差，不做人群身份匹配或客户级归因。

## 模块执行协议

### 对象与基线

- 锁定我方对象、竞品实体、商品/变体、店铺、平台、国家、展示条件和采集时间。
- 记录检索词、发现渠道、纳入/排除理由、覆盖停止条件和未覆盖面。
- 单次快照只能证明当时可见状态；趋势至少需要三个可比快照。

### 信号与机制

- 将价格、促销、排名、评论、内容、广告可见性、库存和渠道信号分别建模。
- 每个变化先检查采集口径、页面实验、地区、登录态和对象版本污染。
- 归因建立原因假设图，区分先行、同步和滞后信号，不以时间相邻证明因果。
- 阈值按平台、品类、生命周期和季节校准，并报告基线中位数与离散度。

### 证据与计算

- 标记 O/M/U/B/E/I，代理信号不得升级为竞品后台销量、花费、利润或库存事实。
- 指标输出公式、分子分母、标准化基准、模型版本和输入/输出哈希。
- 保留反对证据、未变化的关键面、替代解释和可能的采集失败。
- 跨模型版本必须同口径重算，禁止直接比较历史分数。

### 机会与反事实

- 优势判断同时检查来源、稀缺性、可复制性、持续性和我方准备度。
- 机会窗口必须有打开信号、关闭信号、最晚验证日和不行动代价。
- 比较跟随、差异化、观望和退出，不把竞品动作自动转成我方动作。
- 价格变化至少检验清仓、活动、变体切换、缺货和页面展示偏差。

### 失败与闭环

- 采集部分失败时保留已确认事实，范围和归因降级 inconclusive。
- 高置信归因需要至少三个一致且尽量独立的信号；否则给验证任务。
- 事件卡写明 owner、允许用途、禁止用途、责任域、成功、停止和复核日期。
- 下游动作由对应主权域决定；CIM只跟踪是否执行和结果是否支持原假设。
- 到期归因标记 validated/refuted/unresolved，并保留原版本与修订原因。

## extended-competitive-signal-domains 专属决策内核

| 维度 | 本模块要求 |
|---|---|
| 核心机制 | 广告/达人/价格/渠道/供应/口碑只确认外部可见信号 |
| 计算或判定 | 每个域记录可见字段、不可见后台、允许和禁止推断 |
| 主要失效 | 广告数量推花费、缺货标签推真实库存、达人帖推销量 |
| 决策动作 | 生成事件卡路由主权域，不直接产生经营动作 |

本节必须由该模块自己的证据和失败测试验证，不得用通用附录或其他reference代替。
