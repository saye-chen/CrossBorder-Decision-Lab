# 平台、品类与生命周期校准

## 校准原则

基础阈值只作为冷启动线。有效阈值综合平台、品类、生命周期和季节事件；没有足够历史样本时保留“校准未成熟”，使用业务绝对阈值并提高人工复核比例，不制造伪精确系数。

| 校准层 | 必须记录 | 典型影响 |
|---|---|---|
| 平台 | 站点、字段定义、展示条件、刷新节奏、登录/地区差异 | 排名、已售量、广告位和到手价不可跨平台直接比较 |
| 品类 | 客单价、价格弹性、购买频率、评价节奏、退货、季节性 | 决定波动容忍度和确认周期 |
| 生命周期 | 新品、放量、稳态、衰退 | 新品用探索基线，稳态用滚动基线，衰退关注退出信号 |
| 季节事件 | 旺季、大促、节日、天气、平台政策事件 | 优先与历史同期或同组对照比较 |

校准记录包含基础阈值、调整依据或系数、适用对象、生效时间、数据窗口、样本量、模型版本和复核日期。任何系数都应能回指历史波动或业务规则。

## 平台路由

- Amazon：区分自然位/广告位、BSR 类目层级、Coupon/Prime/运费和变体共享评价。
- TikTok Shop：分开商品卡、短视频、直播、达人和付费素材信号；热度不等于稳定购买需求。
- Shopee/Lazada：记录站点、券、跨店优惠、运费补贴和“已售”口径；到手价优先于标价。
- Shopify/DTC：分开站内商品、Meta/Google 广告、邮件、社媒和站点技术变化；公开信号不能证明真实销量。
- Walmart/eBay/Etsy/Temu/Shein：先确认站点展示、卖家/平台经营模式和可得字段，再选择阈值；不照搬 Amazon 口径。

平台规则和市场状态属于动态信息，执行任务时当日核验并记录截止时间。

## 模块执行协议

### 对象与基线

- 锁定我方对象、竞品实体、商品/变体、店铺、平台、国家、展示条件和采集时间。
- 记录检索词、发现渠道、纳入/排除理由、覆盖停止条件和未覆盖面。
- 单次快照只能证明当时可见状态；趋势至少需要三个可比快照。

### 信号与机制

- 将价格、促销、排名、评论、内容、广告可见性、库存和渠道信号分别建模。
- 每个变化先检查采集口径、页面实验、地区、登录态和对象版本污染。
- 归因建立原因假设图，区分先行、同步和滞后信号，不以时间相邻证明因果。
- 阈值按平台、品类、生命周期和季节校准，并报告基线中位数与离散度。

### 证据与计算

- 标记 O/M/U/B/E/I，代理信号不得升级为竞品后台销量、花费、利润或库存事实。
- 指标输出公式、分子分母、标准化基准、模型版本和输入/输出哈希。
- 保留反对证据、未变化的关键面、替代解释和可能的采集失败。
- 跨模型版本必须同口径重算，禁止直接比较历史分数。

### 机会与反事实

- 优势判断同时检查来源、稀缺性、可复制性、持续性和我方准备度。
- 机会窗口必须有打开信号、关闭信号、最晚验证日和不行动代价。
- 比较跟随、差异化、观望和退出，不把竞品动作自动转成我方动作。
- 价格变化至少检验清仓、活动、变体切换、缺货和页面展示偏差。

### 失败与闭环

- 采集部分失败时保留已确认事实，范围和归因降级 inconclusive。
- 高置信归因需要至少三个一致且尽量独立的信号；否则给验证任务。
- 事件卡写明 owner、允许用途、禁止用途、责任域、成功、停止和复核日期。
- 下游动作由对应主权域决定；CIM只跟踪是否执行和结果是否支持原假设。
- 到期归因标记 validated/refuted/unresolved，并保留原版本与修订原因。

## platform-category-lifecycle-calibration 专属决策内核

| 维度 | 本模块要求 |
|---|---|
| 核心机制 | 相同信号按平台、品类、生命周期具有不同基线和滞后 |
| 计算或判定 | 校准记录样本窗、中位数/MAD、事件标签和有效期 |
| 主要失效 | 跨平台硬套、成长期阈值用于衰退期、国家混池 |
| 决策动作 | 未知组合降级并优先建立本地基线 |

本节必须由该模块自己的证据和失败测试验证，不得用通用附录或其他reference代替。
