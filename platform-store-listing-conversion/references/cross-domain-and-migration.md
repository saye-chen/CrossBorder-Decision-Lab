# 跨域合同与六 Skill 迁移

## 交接合同

每次交接含 handoff_id、来源/目标Skill及版本、object_id/page_version、证据截止、claims/evidence/calculations、Gate影响、置信度、allowed/forbidden uses、建议状态、成功/停止/回滚和剩余风险。输入仅 `proposed` 时不得自动生效。

## 主权优先

D05合法安全与产品事实、D06利润、D07真实库存履约是不可补偿约束；D09广告、D11内容、D12营销、D13客户各保留本域动作。D08只决定页面承接、具体页面方案和页面实验。跨域输入失败时仍可报告已观测页面事实，但依赖结论标 `inconclusive`。

## 六 Skill 迁移

- D01：迁入我方关键词承接、页面结构和平台/国家页面应用；保留投资主权。
- D02：共享外部页面快照、变化事件和对象去重；保留竞品发现/确认/归因。
- D07：只读ATP/CTP、配送、退货和容量Gate；保留物流库存模型。
- D09：共享流量承诺、落地页和测量交接；保留预算/出价/归因/放量。
- D11：迁入页面槽位、顺序和生产Brief；保留创意机制和素材生产。
- D13：迁入VOC到页面疑虑/保障映射；保留客户身份、服务、CRM、声誉和CLV。

实施采用复制/重写→等价测试→原域路由→断链验证→删除重复正文。每项记录 source_anchor、target_anchor、owner_before/after、retained_route、parity_test、removal_status。历史报告不改写。
