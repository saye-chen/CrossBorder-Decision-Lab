# 场景输出协议

## 共同底座

当输出包含进入、投资、上架、放量或放弃建议时，至少展示：评估边界、`CIDM-2026.06`、证据截止日、五道门槛摘要、七维总分、置信度、四档决策、红线、关键假设和验证动作。场景协议只改变展示结构，不改变基础模型。

纯监控、数据采集模板或复盘任务若不产生新的投资判断，可以不重复完整评分，但必须引用最近一次评分版本和日期；发生重大变化时重新评分。

## 协议矩阵

| 场景 | 默认交付物 | 必要模块 |
|---|---|---|
| 品类投资决策 | 完整投决报告 | STEP1-STEP8、平台建议、运营打法 |
| 商品链接反查 | 链接反查卡 | 页面身份、产品/流量/评论/利润/供应链/风险、评分 |
| 多候选组合 | 横向决策表 | 硬淘汰、单品排名、最优组合、候补、压力测试 |
| 国家本地化 | 国家进入矩阵 | 消费、平台、法规税务、物流利润、准备度、顺序 |
| 趋势爆品 | 趋势机会清单 | 7/30/90 天、多信号验证、生命周期、追入/退出点 |
| 平台榜单 | 榜单反查表 | 排名持续性、可复制/不可复制、竞争与供应替代 |
| 竞品监控 | 变化与告警摘要 | 新品、价格、库存、评论/销量代理、风险、动作 |
| TikTok 内容 | 内容商品卡 | 3 秒钩子、素材矩阵、达人画像、CPA 约束、疲劳信号 |
| Amazon 长尾 | 关键词机会图 | 关键词簇、首页结构、Review/CPC、新品突破、Listing 改良 |
| Temu 供应链 | 核价生存表 | 目标核价、最低成本线、重量体积、供货与退货风险 |
| 季节/节日 | 选品日历 | 需求启动、下单、到仓、停止补货、清仓节点 |
| 人群需求 | 人群—需求矩阵 | 人群规模、痛点、支付、触达、平台与商品方向 |
| 使用场景 | 问题—方案地图 | 痛点频率、替代方案、商品组合、关联购买、内容方向 |
| DTC/独立站 | 英雄品商业卡 | 价值主张、贡献毛利、CAC/回收、AOV、复购/LTV |
| 利润核算 | 单位经济表 | 完整成本、三场景、ACOS/CPA 临界值、现金占用 |
| 店铺补品 | SKU 缺口矩阵 | 价格带、流量词、关联购买、库存、优先级、蚕食风险 |
| 老品延展 | 变体扩展树 | 原品基线、扩展项、素材/供应复用、增量 ROI、蚕食 |
| 测款验证 | 实验卡 | 假设、样本/预算、周期、通过指标、停止条件、下一步 |

## 输出选择规则

- 用户说“详细、完整、投委会、市场进入”时使用完整报告。
- 用户说“快速、粗看、初筛”时使用快速初筛。
- 用户目标是执行动作而非市场全貌时使用场景交付物。
- 一个任务叠加多个场景时，以主场景协议为骨架，把辅场景和横向模块作为附表，不重复相同内容。
- 多候选超过 5 个时，先输出总览和 Top/Bottom；详细证据放入附表或电子表格，避免正文失控。

## 监控与复盘例外

竞品监控每次输出“发生了什么、为什么重要、是否触发重新评分、下一步动作”。没有变化时只报告监控范围、截止时间和“未触发告警”，不制造结论。

测款复盘按“原假设—实际数据—偏差—结论—下一轮”输出；不得用 GMV 代替净贡献，也不得把样本不足写成验证成功。
