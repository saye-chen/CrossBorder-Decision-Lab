# 使用指南

## 选择输出模式

| 用户目标 | 默认输出 |
|---|---|
| 值不值得进入、市场进入、详细研究 | 完整投决报告 |
| 快速判断、初筛、粗看 | 快速初筛 |
| 监控、清单、日历、扩品、核算、组合、实验 | 场景专属交付物 |
| demo、压测、看一下效果、出报告、保存下来 | 本地 Markdown + 对话摘要 |

用户明确要求格式或篇幅时优先遵从，但不能省略红线、假设和决策依据。

## 最小输入

| 任务 | 最少需要 | 缺失时 |
|---|---|---|
| 品类/商品评估 | 品类或商品、目标国家 | 平台可根据国家和商品推断并标注假设 |
| 链接反查 | 可访问链接、ASIN/商品 ID 或页面材料 | 页面受限时请求截图、页面文本或导出 |
| 多品比较 | 候选清单、国家/平台、选择目标 | 没有预算时先做单品排名，不假设组合约束 |
| 预算组合 | 候选基础分、整体/维度置信度、投资额、数量/预算 | 缺少基础分时先逐个运行核心模型 |
| 国家比较 | 产品、候选国家或区域、履约模式 | 区域请求先选择 2-4 个具体国家 |
| 未知国家 | 目标国家、产品、平台或销售模式 | 使用任意国家本地化路由，联网核验当前官方来源 |
| 未知平台 | 平台名称、国家、产品或类目 | 使用任意平台路由，先判断平台类型和流量逻辑 |
| 店铺补品 | 店铺/SKU 清单、平台、国家 | 无店铺数据时只能提供数据采集模板 |
| 老品扩展 | 原品链接或经营数据、扩展目标 | 无原品数据时输出假设和补数清单 |
| 老品诊断 | 原品链接、最近 4-12 周经营变化 | 缺少经营数据时先按公开信号和补数清单诊断 |
| 竞品监控 | 竞品 ASIN/链接、站点、监控周期 | 缺少历史快照时先建当前快照和触发条件 |
| 上市后巡检 | SKU/ASIN、阶段、KPI 或经营数据 | 缺少真实经营数据时不得伪造放量结论 |
| 退出复盘 | 原始假设、实验/经营数据、损失口径 | 缺少原始假设时先重建假设并标注低置信度 |
| VOC/竞品情报 | 品类/产品、国家/平台、决策问题或评论/竞品材料 | 先建数据源矩阵，不伪造用户原话 |
| 利润核算 | 售价、产品/包装、物流履约、平台费 | 未知项用区间并做压力测试 |
| 测款计划/复盘 | 产品、平台、预算或上限；复盘另需事先门槛与实际数据 | 用相对阶段设计；无事先门槛时只做描述分析，不伪造通过线 |

只有当缺失信息会改变市场、监管属性、平台或投资姿态时才暂停询问；其余情况使用显式假设继续。

## 文件与数据

支持直接文本、链接、截图、Markdown、CSV 和 Excel 等用户提供材料。源文件只读；需要结构化处理时使用任务临时副本。

用户要求文件交付、demo、压测、完整报告、看一下效果或保存下来时，默认生成本地 Markdown；未指定目录则保存到 `$HOME/Downloads`，并在最终回复提供可点击路径。

## 结果解读

- 总分表达基础投资质量，不保证销售结果。
- 置信度表达证据完整性；低整体置信度或核心维度不确定的高分，只能支持轻量验证。
- 门槛红线可以推翻总分。
- 组合效用、本地化准备度和平台适配指标是补充判断，不替代 100 分基础模型。
