# 任意平台路由

## 使用场景

当用户指定的平台不在既有 playbook 中，或平台规则可能变化时，使用本文件。不要因为平台不熟悉就套用 Amazon、TikTok 或 Temu 的逻辑。

## 平台类型

先判断平台属于哪类；一个平台可同时具备多种属性，但必须选择主流量逻辑。

| 类型 | 流量逻辑 | 典型证据 | 常见风险 |
|---|---|---|---|
| 搜索货架电商 | 用户主动搜索和比价 | 关键词、排名、评论、价格、广告位 | 评论壁垒、CPC、价格战 |
| 内容电商 | 内容发现和达人转化 | 素材、达人、评论购买意图、转化链路 | 播放不转化、素材疲劳、佣金侵蚀 |
| 低价/托管平台 | 核价、供给效率、活动 | 核价、动销、退货、供货速度 | 无利润、同质化、履约罚则 |
| 独立站/DTC | 付费流量和品牌信任 | CAC、CVR、AOV、复购、邮件/SEO | CAC 回收失败、信任不足 |
| 批发/B2B | 采购需求和账期 | MOQ、询盘、认证、样品、账期 | 回款慢、定制复杂、合规资料 |
| 二手/收藏平台 | 稀缺性和卖家信誉 | 成交记录、状态、信誉、争议 | 售后争议、假货/IP、价格透明 |
| 社群/私域 | 信任和复购 | 人群、内容、社群互动、复购 | 规模有限、运营重、数据偏样本 |

## 八项平台检查

| 模块 | 必查问题 |
|---|---|
| 账号准入 | 主体、资质、保证金、品牌、类目权限 |
| 流量机制 | 搜索、推荐、达人、活动、广告、站外 |
| 费用结构 | 佣金、支付、广告、达人、仓配、活动、退款 |
| 履约售后 | 仓配模式、时效、退货、赔付、客服、绩效 |
| 内容/Listing | 标题、图片、视频、详情、A+、直播、短视频 |
| 竞争结构 | 头部品牌、价格带、评论/销量壁垒、新品突破 |
| 数据可得性 | 后台指标、公开代理、第三方工具、采样偏差 |
| 平台风险 | 禁售、限流、扣分、下架、资金冻结、申诉 |

## 平台输出

1. 平台类型和流量逻辑。
2. 目标品类在平台上的需求证据。
3. 竞争结构和可进入性。
4. 费用、履约和单位经济。
5. 内容/Listing/广告打法。
6. 平台风险与人工复核项。
7. 测款指标、放量条件和停止条件。

## 禁止套用

- 不把 Amazon 的 Review/CPC 逻辑套到内容电商。
- 不把 TikTok 的播放量当成搜索电商需求。
- 不把 Temu 的核价逻辑套到 DTC。
- 不把独立站毛利率当成 CAC 可回收。
- 不把平台可上架当成法律合规。
