# 错误处理与降级规则

## 降级等级

| 等级 | 条件 | 允许输出 |
|---|---|---|
| A 完整证据 | 核心维度有官方/一手与交叉信号 | 正常评分与完整决策 |
| B 代理证据 | 核心市场数据部分依赖可解释代理 | 评分＋中等置信度＋验证动作 |
| C 假设驱动 | 成本、需求或竞争关键项缺失 | 区间/条件判断，最高默认小规模测试 |
| D 不可判断 | 产品边界、国家、监管属性或页面事实缺失且不同解释会推翻结论 | 暂缓评分，明确请求最少补充信息 |

## 常见失败

- 页面无法访问：尝试规范链接、目标国家页面或用户授权浏览器；仍失败则请求截图、页面文本或导出，不猜测。
- 搜索量/销量不可得：使用关键词簇、排名、评论增长、榜单和内容等代理，注明不能换算精确销量。
- 数据源冲突：检查国家、平台、时间、变体和含税口径；不能消解时展示区间与两种结论的敏感性。
- 供应商无报价：给出目标成本上限和询价清单，不输出“实际净利”。
- 法规属性不明：停止进入建议或降级为条件判断，要求成分、用途、宣称、年龄和技术规格。
- 工具或脚本失败：保留原输入，记录命令与错误摘要；能安全手工复核时继续，否则报告未完成项，不伪造结果。
- 同名交付物：暂停写入并询问覆盖、版本号或改名，不自行决定。
- 清理失败：保留错误证据并报告精确残留路径；不得为“整洁”删除无法确认归属的内容。

## 部分完成

任务包含多个候选或国家时，单个页面/数据源失败不应抹掉已完成结果。分别标记 `完成 / 代理完成 / 待补数 / 无法判断`，只对有足够证据的对象评分。排名时不得把未评分对象当作低分对象。

## 恢复顺序

1. 确认原始请求和输出范围未被新指令替换。
2. 检查并清理已确认属于中断任务的临时目录和进程。
3. 验证源文件仍只读、已有交付物未被部分覆盖。
4. 从最后一个已验证阶段继续，不重复写入或重复抓取无必要数据。
5. 结束时运行自检并披露仍未恢复的部分。
