# 品类投资决策

## 目标

默认用中文为中国跨境卖家输出可执行的投资判断。不要停留在市场概览；用可追溯证据、明确假设、统一评分和商业推理回答：市场够不够大、能否进入、能否赚钱、风险是否可控、凭什么能赢。

只在用户明确要求英文时输出英文。用户未要求文件时直接在对话中交付；要求本地文件时遵循“工作空间与文件规则”。

## 执行模式

- 用户要求市场进入、品类投资、完整报告或详细分析时，输出完整报告，严格覆盖 STEP1-STEP8。
- 用户说“快速判断、初筛、粗看、简版”时，输出快速初筛版。快速版可跳过 STEP1-STEP8 的逐章展示，但不能跳过五道门槛、评分、风险、假设和下一步验证。
- 用户要求监控、链接反查、趋势清单、季节日历、店铺补品、老品扩展、利润核算、组合选择或测款计划时，读取 [references/scene-output-protocols.md](references/scene-output-protocols.md)，使用对应场景交付物，不机械输出完整 STEP1-STEP8。
- 场景交付物若给出进入/投资建议，仍必须展示总分、置信度、四档决策、门槛红线和验证动作；底层继续使用原模型。
- 只有用户要求或原始数据适合建模时，才额外生成 Excel、CSV 或数据透视表。

开始前确定：产品边界、目标国家、目标平台、卖家资源约束、预期价格带和时间窗口。缺失信息若不会实质改变方向，使用显式假设继续；若不同解释会推翻结论，先询问用户。

## 常用使用方式

- 品类进入：`评估宠物饮水机是否值得进入美国 Amazon，给我完整报告。`
- 快速初筛：`快速判断这款露营灯值不值得做德国站。`
- 链接反查：`拆解这个 ASIN/商品链接，分析流量、利润、供应链和风险。`
- 多品组合：`比较这 8 个候选，预算 5 万，只选 3 个。`
- 国家比较：`比较这款产品进入美国、日本和德国，给出进入顺序。`
- 趋势/内容：`找近 90 天美国 TikTok Shop 正在增长且可演示的产品方向。`
- 店铺与老品：`分析我的店铺 SKU 缺口` 或 `给这个老品设计变体扩展方案。`
- 测款：`为这个候选设计 12 周测款、放量和停止条件。`

输入不完整时先使用 [references/usage-guide.md](references/usage-guide.md) 的最小输入清单；批量数据按 [references/input-schemas.md](references/input-schemas.md) 规范读取。

## 工作空间与文件规则

### 源文件

- 将用户提供或指定的源文件视为只读：不覆盖、不改名、不移动、不删除，也不修改文件元数据。
- 需要转换、清洗或分析时，将副本放入本次任务临时目录；最终结果另存为新交付物。
- 不把用户源文件、卖家后台数据、抓取页面或业务报告复制到 Skill 目录。Skill 目录只存放 `SKILL.md`、`agents/`、`references/`、`scripts/` 和必要的 `assets/`。

### 临时工作目录

- 只有任务确实需要中间文件时才创建：`${TMPDIR:-/tmp}/category-investment-decision/<YYYYMMDD-HHMMSS>-<task-slug>/`。
- 优先运行 `python3 scripts/workspace_manager.py create --slug <task-slug> --task-id <task-id>` 创建目录；结束时运行 `python3 scripts/workspace_manager.py cleanup <path> --task-id <task-id>`。
- 在目录中建立 `.task-owner.json` 任务标记，记录任务标识和创建时间；所有截图、抓取页、缓存、计算中间文件和测试导出只放在该目录。
- 结束前停止本任务启动的后台进程或会话，删除整个任务临时目录并验证目录不存在。只删除本任务明确创建的内容，禁止使用可能波及用户文件或其他任务的宽泛清理命令。
- 清理失败时不得静默结束；在最终回复中列出残留路径和失败原因。

### 中断与恢复

- 任务恢复、重试或下一次继续执行时，先检查已记录的本任务临时目录和后台进程；确认属于已中断任务后停止进程并清理残留，再继续工作。
- 已知中断任务标识时，可运行 `python3 scripts/workspace_manager.py cleanup-task --task-id <task-id>`；工具只删除标记与任务标识完全匹配的目录。
- 只清理带有本 Skill 任务标记且能确认归属的目录。无法确认归属、可能仍被其他任务使用或位于临时根目录之外的内容不得删除。
- 中断发生在无法执行清理的阶段时，在能够继续运行的第一个机会完成补清理。

### 最终交付与命名

- 用户未指定目录时，将最终交付物保存到 `$HOME/Downloads`；用户指定目录时使用其指定位置。Skill 目录不得存放业务交付物。
- 默认文件名：`<国家或地区>_<平台>_<品类或对象>_<场景>_<YYYY-MM-DD>.<扩展名>`。
- 链接反查的“对象”优先使用 `ASIN-<编号>`、商品 ID 或品牌加短品名；多候选组合使用 `候选组合`；缺失字段使用明确短标签，禁止使用 `未命名`、`新文件`、`final` 等模糊名称。
- 命名示例：`美国_Amazon_洗发水_品类投资决策_2026-06-22.md`、`美国_Amazon_ASIN-B0XXXX_商品链接反查_2026-06-22.md`、`美国_多平台_候选组合_多候选组合决策_2026-06-22.xlsx`。
- 将 `/ \\ : * ? " < > |` 等不安全字符替换为连字符，合并连续空格和分隔符，并控制文件名长度，避免截断扩展名。
- 写入前检查目标路径。发现同名文件时必须先询问用户，由用户选择覆盖、使用 `_v2`/`_v3` 版本号，或更换名称；未经确认不得覆盖，也不得擅自生成版本文件。
- 最终回复提供交付物的可点击绝对路径；只保留最终交付物和用户源文件，不遗留临时副本。

## 场景路由

先识别用户的主要选品场景，再执行核心模型。读取 [references/selection-scenarios.md](references/selection-scenarios.md)，选择一个主场景；确有必要时增加一个辅场景。若无法判断，默认使用“品类投资决策”。

商品链接反查、多候选组合和国家本地化属于横向模块，可以相互叠加，也可以与主/辅场景同时启用；它们不占主场景与辅场景的名额。例如“比较 5 个 Amazon US 链接并在 5 万预算内选 3 个”应同时启用链接反查、组合决策和美国本地化。

场景层只能调整取证优先级、补充指标、额外输出和测款方式，不得改变以下核心架构：

- 五道前置门槛及红线优先原则；
- 七维权重、0-10 分锚点和加权公式；
- 四档决策阈值与置信度约束；
- 完整报告的 STEP1-STEP8 商业逻辑。

不要把“长期价值乘法链”直接计算成分数。把需求、点击、转化、复购、利润、供应稳定和合规作为验证链；任何一环薄弱都要转化为风险、条件或实验。

## 数据与证据工作流

1. 当前市场、价格、竞品、趋势、平台政策、法规、物流费用等必须联网核验。
2. 优先使用官方监管/平台规则、真实商品页和用户提供的卖家工具数据；行业文章只作补充。
3. 搜索量、销量、广告强度等若无法从公开来源直接验证，必须写成代理信号或估算，不能伪装成精确事实。
4. 每项关键判断标注为“观察数据 / 计算结果 / 分析假设”，记录来源、日期和证据等级。
5. 关键结论尽量由两个独立信号交叉验证。无法验证的数据转化为风险、区间或下一步验证动作。
6. 不只罗列数据；每项证据必须连接到评分、门槛或行动建议。

## 能力边界

- 公开网页、榜单和第三方工具只能提供观察或代理信号，不能冒充卖家后台的精确搜索量、销量、CPC、转化率或广告花费。
- 登录、地区、验证码或反爬限制导致页面不可访问时，使用用户授权的浏览器状态、截图、页面文本或工具导出；没有证据时不得猜测页面内容。
- 未取得真实供应商报价、包装尺寸、履约费和税费时，利润只能输出假设区间与敏感性，不得包装成实际利润。
- 合规、税务、海关、认证和 IP 结论仅用于商业初筛；高风险事项必须由目标国家专业人士、检测机构或律师复核。
- 不执行虚假库存、伪造订单、刷评、侵权跟卖、绕过平台限制或其他违规验证方式。
- 数据不足以支持评分时降低置信度或暂缓评分，不为保持报告完整而编造数字。

## 前置五道门槛

评分前先判断五道门槛。门槛红线优先于总分。

| 门槛 | 核心问题 | 判断要求 |
|---|---|---|
| 市场够大吗 | 需求能否支撑目标规模 | 看关键词簇、趋势、销量/排名代理信号和购买频次；不以单一峰值下结论 |
| 进得去吗 | 新卖家能否获得曝光 | 看首页评论结构、品牌集中度、上架时间、近期新品突破和广告压力 |
| 能赚钱吗 | 完整成本后是否有净利 | 计入产品、包装、关税、头程、履约、佣金、广告、促销、退货、仓储、质检和损耗，并做敏感性分析 |
| 能卖吗 | 合规与 IP 是否可控 | 核验品类适用法规、标签/宣称、平台规则，并提醒专利 FTO、商标与版权检索 |
| 凭什么是你 | 是否有可落地切入楔子 | 从 1-3 星评论、QA、社媒和产品结构中找反复痛点，落到产品、套装、材料、尺寸、包装、说明书、服务或内容 |

默认警戒线只能作为基准，不是跨品类硬规则：

- 美国 Amazon 普通低客单标品，关键词簇总需求偏低且主词月搜索量低于约 1 万时，不建议重仓；高客单、强复购、专业刚需或站外流量可构成补偿。
- 稳态目标净利润率默认应高于 20%；战略引流品、订阅复购品或高 LTV 产品可放宽，但必须展示回收周期和验证路径。
- 首页几乎全是 2,000+ 评论老品且看不到近 12 个月新品突破时，降低竞争可进入性得分，优先寻找细分长尾。
- 高概率不可控的合规、安全或 IP 风险可直接降级为“不建议进入”。
- 没有明确切入楔子时，只能观察、内容测款或轻库存测试。

## 评分与决策

当前核心模型版本：`CIDM-2026.06`。每份报告记录模型版本、证据截止日、目标国家/平台、币种和含税/未税口径；模型升级不得回写或悄然改变旧报告分数。

使用七维 100 分模型：市场需求 20、竞争可进入性 20、利润空间 20、内容传播 10、供应链可控性 10、风险可控性 10、机会窗口 10。

每维先打 0-10 原始分，再计算：

`加权分 = 原始分 ÷ 10 × 权重分值`

风险可控性得分越高表示越安全。总分保留一位小数，评分依据必须引用具体证据或假设，不允许先定结论再反推分数。

| 总分 | 默认决策 |
|---:|---|
| 80-100 | 建议进入 |
| 65-79.9 | 谨慎进入 / 小规模测试 |
| 50-64.9 | 仅观察或内容测款 |
| <50 | 不建议进入 |

若门槛红线导致最终建议低于分数档位，明确写出覆盖原因。数据置信度低时，不得仅凭高分建议重仓。

## STEP1-STEP8

### STEP1：市场需求分析

分析关键词簇、趋势与季节性、销量/排名信号、购买动机、使用场景、复购和配件扩展；输出市场需求得分及进入时机。

### STEP2：竞争分析

分析头部评论壁垒、评分质量、Listing 成熟度、品牌集中度、价格带、广告压力和新品突破；输出竞争可进入性得分，并说明哪类新进入者可能赢。

### STEP3：利润分析

给出售价、完整单位经济模型、贡献利润、净利润率、盈亏平衡 ACOS/CPA、库存与现金周转。至少设置基准、压力、乐观三个场景；输出利润得分及规模化条件。

### STEP4：内容传播分析

分析前后对比、痛点可视化、前三秒钩子、达人/联盟适配、UGC 复制性、礼品或身份表达；输出内容传播得分和 3-5 个具体内容角度。

### STEP5：供应链分析

分析供应商深度、MOQ、交期、定制空间、包装物流、关键质检点、售后和瑕疵；输出供应链可控性得分及控制点。

### STEP6：风险分析

具体核验安全、认证、监管、标签和营销宣称、平台政策、IP、退货、季节性及评价风险；输出风险可控性得分，并单列可推翻结论的红线。不要把“需要进一步确认”写成“无风险”。

### STEP7：机会分析

从未满足人群、差评/QA/社媒痛点、价值缺口、平台错配和时间窗口中识别机会；输出机会得分。切入楔子必须落到可生产或可执行改变。

### STEP8：最终决策

综合门槛、总分和置信度，只选择四档结论之一，并说明推荐投资姿态、进入条件、停止条件和最小验证方案。

## 平台与运营

用户指定平台时只围绕该平台；未指定时基于目标国家选择最相关的 2-4 个平台，不机械罗列所有平台。平台结论可以不同于整体结论。

运营方案必须匹配决策档位，至少包含：目标人群、价格带、首发 SKU、切入楔子、内容角度、流量策略、供应商/质检要求、测试预算或里程碑、停止条件，以及明确不建议做的动作。

## 版式与质量

- 表格控制在 4-6 列，只放短字段；推理放在表格下。
- 完整报告在摘要、评分后、风险和运营方案至少各使用一个优先级色块；STEP1-STEP8 每步只在确有关键判断时使用色块，避免装饰性堆叠。
- 本地 Markdown 可使用模板中的 HTML 色块；聊天中若 HTML 渲染不稳定，使用简洁引用块和 `1级/2级/3级关键信息` 标签。
- 完整报告必须包含总分、四档决策、五道门槛、STEP1-STEP8、风险、运营方案、假设与数据缺口、来源。
- 禁止使用“有潜力、竞争激烈、提升质量、做品牌、需要差异化”等空泛结论，除非紧跟证据、商业含义和具体动作。
- 严格执行“工作空间与文件规则”，交付前完成临时目录和中断残留清理检查。

## 执行结束自检

交付前逐项检查；任何必检项失败都先修正，无法修正则在结论中明确披露：
