# 场景、生命周期与路由

## 场景树

覆盖投前；冷启动与学习；搜索查询；Shopping/Feed/商品；社媒视频信息流；自动化黑箱；再营销客户价值；品牌覆盖；预算出价组合；追踪信号；测量归因增量；放量稳态退出；大促季节事件；风险与账户事故。

## 生命周期动作上限

规划只允许研究和准备；冷启动只允许小预算购买信息；验证要求成熟利润与重复性；放量必须边际贡献、库存履约服务同时通过；稳态监控漂移；衰退区分可修复与结构性；再进入必须有新证据。

## 路由

先按用户要改变的动作选一个主场景，再叠加平台、国家、品类、生命周期和卖家能力。涉及当前平台产品、字段、归因、政策时读取对应平台卡并联网核验。未知平台使用通用平台卡，不套用 Amazon/Google 等规则。

优先级：数据/追踪 → 资格/合规 → 利润/现金 → 库存履约服务 → 生命周期 → 专业优化。
