# 视频商业决策

## 商业门槛

在讨论放量前判断：

1. 利润：完整成本后贡献利润是否为正。
2. 合规/IP：产品和内容宣称是否存在否决项。
3. 供应链：起量后的交期、库存和质检是否可承受。
4. 退货：视频承诺是否可能放大预期落差与退款。
5. 产品-视频匹配：格式、价格带、人群、卖点和差异是否匹配。

门槛不通过时，不因内容评分高而建议放量。

## 电商漏斗

分析曝光 → 播放 → 有效观看 → 商品点击 → 加购 → 下单 → 签收 → 不退货。前三至四个环节可能受视频直接影响；Listing、价格、优惠、物流和评论基础需单列为视频外因素。

检查产品出现时机、证明强度、卖点清晰度、价格锚定、CTA 路径和承诺一致性。不要使用固定产品露出比例作为通用门槛；用同品类有效素材或账号基线校准。

## 单位经济

有输入时计算：

- 单件贡献利润 = 售价 - 产品 - 包装 - 物流 - 佣金 - 支付 - 退货准备金及其他变动成本；
- 可承受最高 CPA = 单件贡献利润 - 售价 × 目标净利润率；
- 盈亏平衡广告费率 = 单件贡献利润 ÷ 售价；
- 批次盈亏平衡订单 = 固定素材、达人和投流成本 ÷ 单件净贡献。

优先复用 CIDM 的 `profit_model.py` 与 `reverse_funnel.py`。缺少成本或漏斗输入时输出反推门槛，不用行业均值伪造精确利润。

## ROI

分别报告收入、贡献利润、净利、ROI 和净利润率。声明 ROI 分子采用归因收入还是归因贡献利润：

`ROI =（归因收入或归因贡献利润 - 内容与推广投入）÷ 内容与推广投入`

`净利润率 = 净利 ÷ 收入`

压力、基准、乐观场景必须明确曝光、CTR、CVR、签收、退货、单位贡献和投入来自 `M/U/B/I` 中哪一类。输入不足时只给敏感性或反推门槛。

## 结论

- `Go`：门槛通过且单位经济有安全垫，进入受控测试或放量。
- `Iterate`：内容或商业链存在可修复断点，给出一次或有限次数迭代。
- `Stop`：存在合规否决、负贡献、不可控退货或无法修复的产品-视频错配。

每个结论包含预算、指标、样本、复盘时间和停止条件。
