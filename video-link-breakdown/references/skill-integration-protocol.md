# VLB 跨域集成协议

VLB 拥有素材观察、内容机制、产品一致性、迁移性和生产实验主权；不拥有竞争事实、客户增量、广告预算、页面承接或投资主权。

输入/输出卡必须含 `object_id`、素材版本、平台/国家、`as_of_time`、证据 ID、授权状态、机制假设、迁移等级、置信度、允许用途、禁止用途和生产成本。向 CIM 交换变化与机制，向 CIG 交换去识别人群任务，向 D08 提交页面可用内容，向 D09 提交付费素材候选，向 CIDM 只提交 `proposed` 投资输入。

禁止把播放、内容评分或付费胜出直接写成销量、客户增量或可复制机制。部分失败时保留逐帧事实，机制与商业结果降级；冲突按对象版本、证据等级、授权和主权裁决，不多数投票。回写包含责任人、成功、停止、回滚与结果回填。
